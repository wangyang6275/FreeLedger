## Executive Summary

ColorFuLedger 是一款面向个人用户和个体工商户的纯本地记账应用，支持 iOS 和 Android 双平台（各自原生开发）。它通过完全离线的架构设计从根本上解决用户的记账隐私问题，以“3 秒极速记账”为核心体验，提供智能分类排序、多货币记账、首发中英双语界面、数据可视化报表、灵活数据导出和主题风格切换。完全免费、无广告，专注于做全球最好用的免费离线记账工具。

> *“你的钱，你的数据，你的自由。”*

**MVP 目标用户：个人用户。** 个体工商户功能（商户模式、发票管理等）作为增长路径在 P2 版本实现。

## Core Vision

### Problem Statement

当前市面上的记账应用普遍存在三大痛点：收费门槛高、用户隐私保障差（数据上传云端）、广告泛滥影响体验。用户的财务数据是最敏感的个人信息之一，但大多数记账应用要求联网并将数据存储在第三方服务器上，用户对自己的数据缺乏真正的控制权。

### Problem Impact

- 个人用户被迫在隐私和便利之间做出妥协
- 个体工商户的经营财务数据面临泄露风险
- 免费应用充斥广告，付费应用增加使用成本
- 换手机或设备丢失时，云端数据归属不明确
- 用户无法自由掌控自己的财务数据导出和使用
- 跨国/跨币种用户缺乏离线多货币记账方案
- 纯本地应用用户面临设备丢失/损坏时数据全部丢失的风险，且用户往往忘记手动备份

### Why Existing Solutions Fall Short

现有记账应用虽然功能丰富，但普遍依赖云端同步模式，用户数据被上传至第三方服务器。免费版本附带大量广告，核心功能被锁在付费墙后。导出功能受限，用户无法自定义导出内容和格式。记账流程往往步骤繁琐，无法在 3 秒内完成一笔记录。多数应用的多语言和本地化支持不完善，数字格式、日期格式等未能适配不同地区用户习惯。

**主要竞品分析：**

- **随手记**：云端同步，广告多，付费功能多，多货币支持弱
- **钱迹**：最接近的竞品（本地优先、基本免费），但有少量广告，不支持多货币，仅支持中文
- **MoneyWiz**：多货币和多语言最强（20+语言），但需付费订阅，依赖云同步
- **Money Manager**：本地为主但有广告，UI 朴素，导出功能基础

**开源竞品：**

- **GnuCash**：老牌开源财务软件，功能强大但 UI 极其复古，移动端体验差
- **Money Tracker / Buckwheat**：开源记账 App，但社区小、更新慢、功能简陋

开源竞品的共同弱点是 UI/UX 差、移动体验差、缺乏打磨感。ColorFuLedger 的差异化恰好是——**开源品质，商业级体验**。

没有任何竞品能同时做到「纯本地 + 完全免费 + 极速记账 + 多货币 + 多风格主题」的组合。

### Proposed Solution

ColorFuLedger 采用 iOS (Swift) 和 Android (Kotlin) 原生开发，配合 SQLite 本地数据库，确保极致的平台体验和性能。所有数据完全保存在用户设备上，从架构层面消除隐私隐患。

#### MVP 核心特性

- **3 秒极速记账**：打开 → 输入金额 → 选分类 → 完成。智能分类根据使用习惯自动排序
- **多货币支持**：货币作为记账条目的基本属性（与金额、分类并列），使用 ISO 4217 标准货币代码存储，支持手动汇率设置，报表按用户主货币汇总。记账的本质是记录事实，不依赖实时汇率
- **多语言架构**：首发中文/英文，语言包内置 App 内离线可用，数据层语言无关（分类用 key 存储），采用 i18n 国际化框架可快速扩展更多语言
- **本地化格式**：数字、日期、货币符号通过系统 Locale API 自动适配用户地区设置，零额外实现成本
- **标签/项目分组**：支持按场景分组记账（如“东南亚旅行”、“装修”、“双11”），可按标签筛选和汇总
- **智能备份体系**：支持加密备份文件导入/导出 + 定期备份提醒，数据不经过任何服务器
- **数据可视化**：月度收支饼图、趋势折线图、标签分组柱状图，直观展示财务状况
- **灵活数据导出**：MVP 支持 CSV 格式，用户可自定义导出内容顺序
- **主题系统**：1 套默认精品主题 + 主题切换架构预留，后续版本扩展更多风格
- **完全免费无广告**：无付费墙、无广告、无订阅

#### 未来规划（P2）

- **双角色模式**：个人模式（日常收支）+ 商户模式（含基础发票管理和收支报表），MVP 数据模型预留 account_type 字段
- **发票管理**：个体工商户基础发票手动录入
- **税务周期报表**：按月/季/年生成汇总报表，随商户模式一起上线
- **精品主题扩展**：增加 2-3 套完整 UI 风格（极简风、少女风、科技风等）
- **PDF 导出**：支持 PDF 格式导出
- **自动本地备份**：自动备份到本地文件系统（iOS Files / Android Documents）
- **账本导入合并**：支持多设备间手动合并账本数据
- **按年份数据归档**：旧数据归档不影响日常使用速度

### Key Differentiators

1. **隐私优先架构**：纯本地存储，零网络依赖，用户数据绝不离开设备
2. **极致效率**：3 秒完成记账，智能分类越用越快
3. **真正免费**：无任何商业变现设计，专注用户体验
4. **安全备份**：加密本地备份方案兼顾隐私承诺和数据安全
5. **角色模式切换**：个人和商户两种 UI 模式，各取所需不相互干扰
6. **精品主题体验**：完整 UI 风格切换，不只是颜色变化
7. **国际化友好**：多货币记账 + 多语言界面 + 本地化数字/日期格式，架构预留 RTL 和更多语言扩展
8. **离线国际化零矛盾**：基于第一性原理设计——记账是记录事实、语言是界面皮肤、格式化由 OS 托管，离线架构与国际化完全兼容，无需任何网络依赖

**一句话定位：全球最好用的免费离线记账应用——隐私优先、三秒记账、多币种、多风格。**

## Target Users

### Primary Users

#### 画像 1：小林 — 精打细算的职场人
- **年龄：** 25 岁，互联网公司运营
- **背景：** 月薪到手后总觉得"钱不知道花哪了"，想存钱买车但每月月光
- **记账习惯：** 每笔消费随手记，午饭、奶茶、打车都不放过
- **核心需求：** 知道钱花哪了、月底有没有超支、分类预算和超支提醒、离买车目标还有多远、基础搜索（快速找到历史记录）
- **现状痛点：** 试过随手记但广告太多，试过钱迹但想要多货币（偶尔海淘），不愿为记账付费
- **"aha 时刻"：** 月底打开饼图发现奶茶竟然花了 800 块——"原来钱都花在这了！"

#### 画像 2：陈叔 — 稳健理财的中年人
- **年龄：** 48 岁，企业中层管理
- **背景：** 收入稳定，但家庭开支复杂（房贷、孩子学费、父母赡养费），想清楚掌握家庭财务全貌
- **记账习惯：** 晚上睡前统一记录当天所有开支，固定支出多（房贷、物业、学费）
- **核心需求：** 月度/年度财务趋势、重复记账模板（固定支出一键记录）、距离财务自由还有多远、各项开支占比
- **现状痛点：** 极度注重隐私，不信任云端同步，不愿把家庭财务数据交给任何第三方；每月重复录入固定支出太繁琐
- **"aha 时刻"：** 看到年度趋势折线图，发现家庭支出连续 3 个月下降——"理财计划生效了！"

#### 画像 3：Amy — 全球化生活的年轻人
- **年龄：** 22 岁，海外留学生
- **背景：** 在英国读书，生活费是英镑，父母打人民币过来，偶尔欧洲旅行用欧元
- **记账习惯：** 消费时随手记，每周看一次报表
- **核心需求：** 多货币记账、汇率快照自动保存（每笔消费关联当时的汇率）、"这个月爸妈给了多少（CNY）我花了多少（GBP）"的清晰对照
- **现状痛点：** 现有 App 要么不支持多货币，要么需要联网同步汇率，而且大多只有中文或英文界面不够友好
- **"aha 时刻"：** 给爸妈导出一份 CSV——"这个月我花了这些，下个月能省 200 镑"

### Secondary Users

MVP 阶段暂不专门服务，但架构预留支持：

## MVP Scope

### Core Features

MVP 聚焦个人用户的极致记账体验，包含 P0 + P1 功能：

**P0 — 必须有：**
1. **3 秒极速记账** — 打开 → 金额 → 分类 → 完成，智能分类按使用频率排序
2. **多货币支持** — ISO 4217 货币代码，手动汇率，汇率快照保存，主货币汇总
3. **智能备份** — 加密备份文件导入/导出 + 定期备份提醒
4. **完全免费无广告**

**P1 — 应该有：**
5. **多语言（中/英）** — i18n 框架 + 内置双语语言包
6. **本地化格式** — 系统 Locale API 适配数字/日期/货币格式
7. **标签/项目分组** — 按场景分组（旅行、装修等），可筛选汇总
8. **数据可视化** — 月度收支饼图、趋势折线图、标签柱状图
9. **灵活导出** — CSV 格式，自定义内容顺序
10. **主题系统** — 1 套默认精品主题 + 切换架构预留
11. **基础搜索** — 按关键词搜索历史记录

### Out of Scope for MVP

| 功能 | 排除理由 | 预留策略 |

| 发票管理 | 商户功能，复杂度高 | 随商户模式上线 |
| 税务周期报表 | 依赖商户模式 | 随商户模式上线 |

| 重复记账模板 | 便捷功能，非核心 | P2 实现 |
