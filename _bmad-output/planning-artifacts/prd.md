# Product Requirements Document - FreeLedger

**Author:** Wangyang
**Date:** 2026-02-14

## Executive Summary

FreeLedger 是一款纯本地离线记账应用，支持 iOS 和 Android 双平台原生开发。以"3 秒极速记账"为核心体验，通过完全离线架构从根本上解决用户隐私顾虑。完全免费、无广告。

**核心差异化：** 纯本地 + 完全免费 + 无广告 + 极速记账 — 市面上没有竞品能同时做到这一组合。

**目标用户：** 个人用户（精打细算的职场人、注重隐私的中年人、海外留学生）

**技术栈：** iOS (Swift) + Android (Kotlin) + SQLite，零网络依赖

**一句话定位：** *"你的钱，你的数据，你的自由。"*

## Success Criteria

### User Success

| 指标 | 衡量方式 | 目标 |
|------|---------|------|
| **记账习惯养成** | 连续 7 天有记账行为的用户占比 | > 50% |
| **核心功能使用** | 使用数据可视化（报表/图表）的用户占比 | > 40% |
| **记账效率** | 单笔记账平均耗时 | < 3 秒 |
| **进阶功能渗透** | 使用标签等进阶功能的用户占比 | > 20% |
| **数据安全意识** | 至少完成 1 次备份的用户占比 | > 30% |
| **"aha 时刻"触达** | 首次查看报表/图表的用户占比（7 天内） | > 60% |

**核心情感成功：掌控感** — 用户养成记账习惯后，真正感受到对个人财务的掌控。"钱去哪了我清清楚楚"的确定感是 FreeLedger 最核心的用户价值。

### Business Success

| 时间节点 | 目标 |
|---------|------|
| **3 个月** | 100 用户，验证核心价值，收集早期反馈 |
| **12 个月** | 1,000 用户，形成稳定口碑和社区 |
| **长期** | 成为"免费离线记账"品类首选应用 |

| KPI | 目标 |
|-----|------|
| **7 日留存率** | > 40% |
| **30 日留存率** | > 25% |
| **App Store 评分** | ≥ 4.5 |
| **正面评价占比** | > 70% |

### Technical Success

详见 [Non-Functional Requirements](#non-functional-requirements) 中的性能和可靠性指标。核心目标：冷启动 < 2 秒，记账 < 3 秒，10 万条记录不卡顿，零数据丢失。

### Measurable Outcomes

- **MVP 验证通过信号：** 3 个月 100 用户 + 7 日留存 > 40% + 评分 ≥ 4.5 → 推进 P2
- **核心体验验证：** 记账 < 3 秒 + 习惯养成率 > 50% → 核心价值成立
- **技术质量验证：** 10 万条不卡顿 + 零数据丢失 → 架构决策正确

## Product Scope

### MVP Strategy

**MVP 方式：** 问题解决型 — 验证"用户是否需要一个纯离线、免费、极速的记账工具"

**资源需求：** 单人或小团队，双平台原生并行开发

**范围决策：** P0 + P1 全部纳入 MVP，一次性交付完整体验

**预估工期：** P0+P1 共 8-10 周（单平台），双平台并行推进

### MVP Feature Set

**P0 必须有：**

| # | 功能 | 说明 |
|---|------|------|
| 1 | 3 秒极速记账 | 打开 → 金额 → 分类 → 完成，智能分类按使用频率排序 |
| 2 | App 密码锁 | App 级别密码保护，打开时需验证 |
| 3 | 智能备份 | 备份文件导入/导出 + 定期备份提醒（不加密） |
| 4 | 完全免费无广告 | 天然属性 |
| 5 | 记录编辑/删除 | 含删除确认提示 |
| 6 | 新手引导 | 可跳过、可在设置中重新触发或关闭，用户完全掌控 |

**P1 应该有（纳入 MVP）：**

| # | 功能 | 说明 |
|---|------|------|
| 7 | 多语言（中/英） | i18n 框架 + 内置双语语言包 |
| 8 | 本地化格式 | 系统 Locale API 适配数字/日期/货币格式 |
| 9 | 标签/项目分组 | 按场景分组，可筛选汇总 |
| 10 | 数据可视化 | 收支饼图、趋势折线图、标签柱状图 |
| 11 | 灵活导出 | CSV 格式，自定义内容顺序 |
| 12 | 主题系统 | 1 套默认精品主题 + 切换架构预留 |
| 13 | 基础搜索 | 按关键词搜索历史记录 |

### Post-MVP Roadmap

**Phase 2（增长期，6-12 个月）：**
- 预算/限额 + 超支提醒
- 重复记账模板（固定支出一键记录）
- 精品主题扩展（少女风、科技风、商务风）
- PDF 导出 + 自动本地备份
- 商户模式 + 发票管理 + 税务报表
- 生物识别解锁（Face ID / 指纹）

**Phase 3（扩张期，1-2 年）：**
- 更多语言支持（日/韩/西班牙语，社区贡献）
- RTL 语言支持
- 多账本 + 账本合并
- 年度数据归档
- 云端同步（可选，需用户授权）

### Risk Mitigation Strategy

**技术风险：** 低 — 技术栈成熟（Swift/Kotlin + SQLite），无复杂后端架构，无第三方依赖

**市场风险：** 低 — 免费无广告的离线记账有明确用户需求，竞品分析已验证市场空间

**资源风险：** 主要风险在于双平台并行开发的工作量。缓解策略：保持功能一致但允许 UI 差异化，利用各平台原生能力减少自定义开发

## User Journeys

### 旅程 1：小林的极速记账日常（核心成功路径）

**开场：** 小林，25 岁，互联网运营，月薪到手总觉得"钱不知道花哪了"。试过随手记，广告太多；试过钱迹，不支持多货币（偶尔海淘）。她想存钱买车，但每月月光。

**发展：** 在 App Store 搜"免费记账 无广告"，看到 FreeLedger——"你的钱，你的数据，你的自由"。下载后无需注册，直接设置主货币 CNY，看到清爽的主界面。午饭后掏出手机，点"记一笔"→ 输入 25 → 选"餐饮"（已排第一位）→ 完成。3 秒不到，比发微信还快。

**高潮：** 连续记了一周，周末打开报表，饼图显示餐饮占比最高——奶茶竟然花了 800 块！"原来钱都花在这了！" 她给餐饮分类加了"旅行基金"标签，开始有意识地控制。

**结局：** 3 个月后，智能分类越来越准，记账已成为肌肉记忆。她打开趋势折线图，月度支出连降 3 个月。距离买车又近了一步。她把 FreeLedger 推荐给同事："用这个，免费的，不用担心隐私。"

**揭示的能力需求：** 极速记账流程、智能分类排序、数据可视化（饼图/折线图）、标签分组、分享推荐

### 旅程 2：陈叔的隐私堡垒（隐私敏感 + 备份恢复）

**开场：** 陈叔，48 岁，企业中层。收入稳定但开支复杂——房贷、孩子学费、父母赡养费。他极度注重隐私，绝不信任任何云端服务。之前用 Excel 记账，费时费力。

**发展：** 儿子推荐了 FreeLedger——"爸，这个完全离线，数据只在你手机上。" 陈叔首次打开 App，新手引导用 3 步说明了核心操作。他每晚睡前统一记录当天所有开支。分类虽然多，但常用的"房贷""学费""赡养费"很快排到了前面。

**高潮：** 用了半年后，陈叔要换手机。他想起 App 每月弹出的备份提醒，早就在 Files 里存了一份备份。新手机安装 FreeLedger，导入备份文件——所有数据完整还原，一条不少。

**结局：** 陈叔看到年度趋势图，家庭支出连续 3 个月下降——"理财计划生效了！" 他彻底放心了：数据在自己手上，换手机也不怕丢。

**揭示的能力需求：** 新手引导流程、定期备份提醒、备份导出/导入、备份完整还原、年度趋势图、智能分类排序、App 密码锁

### 旅程 3：Amy 的留学生活（国际化用户 + 标签导出）

**开场：** Amy，22 岁，英国留学生。她用英镑生活，需要清楚记录每笔花销向父母汇报。现有 App 要么有广告，要么需要联网。

**发展：** Amy 下载 FreeLedger，将 App 语言切换到英文。在超市买菜记一笔 32.50 日用，给周末巴黎旅行的花费打上"Paris Weekend"标签。她每周看一次报表，了解自己的消费结构。

**高潮：** 月底 Amy 打开报表，饼图清晰显示各分类占比。她按"Paris Weekend"标签筛选，一眼看到旅行花了多少。她导出一份 CSV 发给爸妈——"这个月我花了这些，下个月能省 200 镑。"

**结局：** 室友看到 Amy 的英文界面觉得好用，也装上了。FreeLedger 成了她们留学生圈子里的记账标配。

**揭示的能力需求：** 中英双语切换、标签按场景分组、按标签筛选汇总、CSV 导出、数据可视化

### 旅程 4：新手小王的引导体验（首次使用 + 边缘场景）

**开场：** 小王，28 岁，从未用过记账 App。朋友推荐了 FreeLedger，他抱着"试试看"的心态下载了。

**发展 - 成功路径：** 打开 App，出现简洁的引导页——第 1 页：选择主货币；第 2 页：3 秒演示动画展示"记一笔"的核心流程；第 3 页："开始你的第一笔记账"。小王跟着引导记录了今天的午餐 ¥18，整个过程不到 10 秒。引导完成，主界面出现第一条记录，有成就感。

**边缘场景 - 输入错误：** 小王发现金额输错了，点击记录可以编辑修改。他不小心删除了一条记录，系统弹出确认提示"确定删除？此操作不可撤回"，防止误删。

**边缘场景 - 放弃可能：** 如果小王跳过引导直接进入主界面，界面依然简洁不凌乱，底部有明显的"+"按钮。即使没看引导，3 秒记账的直觉设计让他立刻上手。

**结局：** 小王觉得"这比我想象的简单多了"，决定坚持记账。

**揭示的能力需求：** 新手引导流程（可跳过）、记录编辑/删除、删除确认提示、直觉化 UI 设计、引导动画

---

### 旅程 5：换机用户的数据迁移（备份恢复专项）

**开场：** 小李，用 FreeLedger 记了 8 个月的账，积累了 2000+ 条记录。她的旧 iPhone 摔碎了屏幕，需要紧急换机。

**发展：** 小李想起之前 App 提醒她备份时，她导出了一份备份文件到 iCloud Drive（文件管理）。新 iPhone 到手后，从 App Store 下载 FreeLedger，打开 App 选择"导入备份"，从 Files 中找到备份文件。

**高潮：** 加载进度条走完，所有 2000+ 条记录、分类设置、标签全部恢复。她打开报表，数据和旧手机上一模一样——零丢失。

**边缘场景 - 无备份：** 如果小李从未备份过，旧设备损坏意味着数据全部丢失。这强化了备份提醒功能的重要性——App 应在用户记账达到一定数量后主动提醒首次备份。

**结局：** 小李感叹"还好之前听了提醒做了备份"，从此养成定期备份习惯。

**揭示的能力需求：** 备份导出到系统文件管理、备份导入恢复、备份完整性校验、智能备份提醒（基于记录数量触发）、恢复失败引导

## Domain-Specific Requirements

### 数据安全与隐私

- **本地数据存储：** 不做数据库级加密，依赖 iOS/Android 操作系统级文件保护（App 沙箱）
- **备份文件：** 不加密，纯数据文件导出/导入
- **App 密码锁（P0）：** 支持 App 级别密码保护，用户可设置数字密码，打开 App 时需输入密码。后期可扩展生物识别（Face ID / 指纹）

### 财务数据准确性

- **金额精度：** 使用整数最小单位存储（如分），避免浮点误差
- **计算一致性：** 报表汇总确保精确，避免舍入累计误差

### 范围决策记录

- **多货币支持：移除** — 原 P0 需求，因联网获取汇率与离线架构冲突、手动输入汇率不便捷，决定不做。App 仅支持单一主货币记账
- **云端同步：暂不做** — 保持纯离线架构，后期版本再评估
- **数据加密：不做** — 隐私保障通过离线架构 + App 密码锁实现，不做额外加密层

## Mobile App Specific Requirements

### 项目类型概述

FreeLedger 采用 iOS (Swift) 和 Android (Kotlin) 双平台原生开发，配合 SQLite 本地数据库。纯离线架构，零网络依赖，确保极致的平台体验和性能。

### 平台需求

| 维度 | iOS | Android |
|------|-----|--------|
| **开发语言** | Swift | Kotlin |
| **最低版本** | iOS 16+ | Android 10+ |
| **数据库** | SQLite | SQLite |
| **架构** | 纯本地离线 | 纯本地离线 |
| **分发渠道** | App Store | Google Play |

### 设备权限

按需申请，开发到需要的功能时自然获取对应权限：

| 权限 | 用途 | 触发时机 |
|------|------|--------|
| **文件系统访问** | 备份文件导出/导入 | 用户执行备份操作时 |
| **本地通知** | 备份提醒 | 用户首次使用后触发设置 |

后续版本根据功能需要可能新增权限，遵循按需申请原则。

### 离线模式

- **完全离线设计：** App 全部功能均在无网络环境下正常运行
- **零网络请求：** 不发起任何网络请求，不包含任何网络相关代码
- **数据完全本地：** SQLite 数据库存储在 App 沙箱内

### 本地通知策略

- **备份提醒：** 基于本地调度的定期通知，提醒用户备份数据
  - 首次触发：用户记账达到一定数量（如 50 条）且未备份过
  - 定期触发：每月提醒一次（可由用户在设置中调整或关闭）
- **无推送服务器：** 所有通知均为本地通知，不依赖 APNs/FCM 推送服务

### 商店合规

- **Apple App Store：** 开发者账号已有，遵循 App Store Review Guidelines
- **Google Play：** 开发者账号已有，遵循 Google Play Developer Policy
- **隐私政策：** 需准备隐私政策页面，说明"不收集任何用户数据、不进行网络传输"
- **数据安全声明：** Google Play 数据安全表单需声明"不收集、不共享任何数据"

### 实现注意事项

- **双平台并行开发：** iOS 和 Android 各自原生，保持功能一致性，UI 遵循各平台设计规范（iOS: Human Interface Guidelines, Android: Material Design）
- **本地化：** 利用各平台原生 i18n 框架，中英双语内置
- **性能目标：** 冷启动 < 2 秒，记账操作 < 3 秒，10 万条记录不卡顿

## Functional Requirements

### 记账核心

- **FR1:** 用户可以快速创建一笔收入或支出记录（金额 + 分类）
- **FR2:** 用户可以选择预设的收支分类来归类每笔记录
- **FR3:** 系统可以根据用户使用频率自动排序分类列表
- **FR4:** 用户可以编辑已有的记账记录
- **FR5:** 用户可以删除已有的记账记录（需二次确认）
- **FR6:** 用户可以为每笔记录添加备注信息

### 分类管理

- **FR7:** 系统提供一套默认的收支分类
- **FR8:** 用户可以自定义分类（新增、编辑、删除）
- **FR9:** 用户可以分别管理收入分类和支出分类

### 标签与分组

- **FR10:** 用户可以创建自定义标签（如"旅行""装修"）
- **FR11:** 用户可以为每笔记录打上一个或多个标签
- **FR12:** 用户可以按标签筛选和查看记录
- **FR13:** 用户可以按标签汇总查看收支统计

### 数据可视化

- **FR14:** 用户可以查看月度收支饼图（按分类占比）
- **FR15:** 用户可以查看收支趋势折线图（按月度变化）
- **FR16:** 用户可以查看标签分组柱状图
- **FR17:** 用户可以选择报表的时间范围

### 搜索与筛选

- **FR18:** 用户可以按关键词搜索历史记账记录
- **FR19:** 用户可以按时间范围筛选记录
- **FR20:** 用户可以按分类筛选记录

### 备份与恢复

- **FR21:** 用户可以将全部数据导出为备份文件到系统文件管理
- **FR22:** 用户可以从备份文件导入恢复全部数据
- **FR23:** 系统可以校验备份文件的完整性
- **FR24:** 系统可以在用户记账达到一定数量后提醒首次备份
- **FR25:** 系统可以定期提醒用户进行备份（频率可调整或关闭）
- **FR26:** 用户可以在恢复失败时获得明确的错误提示和引导

### 数据导出

## Non-Functional Requirements

### 性能

| 指标 | 目标 | 说明 |
|------|------|------|
| **App 冷启动** | < 2 秒 | 从点击图标到主界面可操作 |
| **记账操作完成** | < 3 秒 | 从打开到保存一笔记录 |
| **报表加载** | < 1 秒 | 切换到报表页面后图表渲染完成 |
| **搜索响应** | < 0.5 秒 | 关键词搜索返回结果 |
| **数据容量** | 10 万条记录无卡顿 | 所有操作保持流畅 |
| **备份文件大小** | 10 万条记录 < 50MB | 便于存储和传输 |
| **备份/恢复速度** | 10 万条记录 < 30 秒 | 导入导出操作 |

### 安全与隐私

- **NFR-S1:** App 不包含任何网络请求代码，确保零数据外泄可能
- **NFR-S2:** App 密码锁在 App 进入后台后重新激活（可配置宽限时间）
- **NFR-S3:** 本地数据依赖 OS 级沙箱保护（iOS App Sandbox / Android App Sandbox）
- **NFR-S4:** 备份文件不包含 App 密码信息，防止通过备份文件绕过密码保护
- **NFR-S5:** 金额存储使用整数最小单位（分），确保财务计算零误差

### 可访问性

- **NFR-A1:** 遵循各平台基础无障碍标准（iOS: VoiceOver 支持, Android: TalkBack 支持）
- **NFR-A2:** 界面元素提供语义化标签，支持屏幕阅读器
- **NFR-A3:** 文字大小支持系统级动态字体调整
- **NFR-A4:** 交互按钮最小点击区域 44x44pt (iOS) / 48x48dp (Android)

### 可靠性

- **NFR-R1:** 数据写入采用事务机制，确保零数据丢失
- **NFR-R2:** App 异常退出后重新打开，数据保持完整
- **NFR-R3:** 备份恢复后数据 100% 完整还原
- **NFR-R4:** 设备存储空间不足时，App 给出明确提示而非崩溃
